# Security Policy

## Supported Versions

KazInni Assistant is a hosted web application. The latest version running on GitHub Pages is always the supported version.

| Version          | Supported          |
| ---------------- | ------------------ |
| Latest (live)    | :white_check_mark: |
| Older versions   | :x:                |

## Reporting a Vulnerability

We take security seriously at KazInni. If you discover a security vulnerability, please report it responsibly.

### How to Report

- **Email:** kazinni.teleioosavour@gmail.com
- **Subject line:** "Security Vulnerability - KazInni Assistant"

Please include:
- A detailed description of the vulnerability
- Steps to reproduce the issue
- Any potential impact
- Suggested fix (if you have one)

### What to Expect

| Stage | Timeline |
|-------|----------|
| Acknowledgment | Within 48 hours |
| Initial assessment | Within 5 business days |
| Patch release (if confirmed) | Within 14 days |
| Public disclosure | After patch is deployed |

### Disclosure Policy

- Please do **not** publicly disclose the vulnerability before we've had a chance to address it
- We will credit researchers who responsibly report valid vulnerabilities (unless you prefer to remain anonymous)
- We treat all reports with strict confidentiality

### Out of Scope

The following are not considered vulnerabilities:
- Issues with third-party services (Firebase, Font Awesome, Google Fonts, PostImages)
- HTTPS/SSL configuration of GitHub Pages (managed by GitHub)
- Brute force attacks on Firebase Authentication
- Issues requiring physical access to a user's device
